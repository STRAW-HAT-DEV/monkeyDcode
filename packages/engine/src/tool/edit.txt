Edits files using hashline patches (preferred) or exact string replacement (fallback).

## Hashline mode (preferred for existing files)
1. Call `read` first — output includes `[path#TAG]` and `N:line` rows.
2. Pass a `hashline` patch with the same `#TAG` and tight line ranges:
   - `replace N..M:` then `+` body rows
   - `delete N` / `insert after N:` / `insert head:` / `insert tail:`
3. Stale `#TAG` or changed line content → error with re-read hint. Re-read before retrying.
4. Syntax errors are caught **before write** — patch is rejected if the result would not parse.

## Search/replace mode (fallback)
- `oldString` / `newString` with fuzzy matching when hashline is not used.
- Preserve exact indentation from read output (after the `N:` prefix).
- `replaceAll` renames/replaces every occurrence.

Rules:
- Prefer editing existing files; use `write` for new files only.
- Never include the line-number prefix in oldString.
- On hashline stale rejection: STOP and re-read.
